# 클라이언트 요청

일본어에서 한국어로 문서 번역을 했을 때 검수하는 것을 만들고 싶어해

---

# 요청 분석

### 1. 요청 요약
- 클라이언트는 일본어에서 한국어로 번역된 문서를 검수하는 자동화 도구를 개발하고자 합니다.

### 2. 개발 To Do 리스트
- 번역된 문서 업로드 기능 구현 (upload_translation.py)
- 번역 검수 알고리즘 개발 (review_algorithm.py)
- 검수 결과를 사용자에게 보여주는 UI 개발 (review_ui.py)
- 검수 결과 피드백 기능 추가 (feedback_system.py)
- 데이터베이스 설계 및 연동 (database_setup.py)

### 3. 필요한 기술
- Python (주 언어)
- Flask 또는 Django (웹 프레임워크)
- HTML/CSS/JavaScript (프론트엔드)
- SQLite 또는 PostgreSQL (데이터베이스)

### 4. 클라이언트에게 확인해야 할 질문
- 검수할 문서의 형식은 무엇인가요? (예: PDF, DOCX 등)
- 검수 기준이나 알고리즘에 대한 구체적인 요구사항이 있나요?
- 사용자 인터페이스에 대한 선호사항이 있나요?
- 검수 결과를 어떻게 저장하고 관리할 것인지에 대한 계획이 있나요?

### 5. 기본 기능
- 번역된 문서의 업로드 및 저장
- 검수 알고리즘에 따른 자동 검수 기능
- 검수 결과를 사용자에게 표시하는 기능

### 6. 추가 비용 가능 기능
- 사용자 맞춤형 검수 기준 설정 기능
- 다국어 지원 기능 추가
- 검수 결과의 통계 및 리포트 생성 기능

### 7. 주의사항
- 번역 검수의 정확성을 높이기 위해 충분한 테스트가 필요합니다.
- 사용자 데이터 보호 및 보안에 대한 고려가 필요합니다.

# 추천 폴더/파일 구조

## 추천 폴더/파일 구조

```
translation_review_tool/              (프로젝트 루트 폴더)
├─ app/                               (애플리케이션 관련 파일)
│  ├─ __init__.py                     (패키지 초기화 파일)
│  ├─ upload_translation.py            (번역된 문서 업로드 기능 구현)
│  ├─ review_algorithm.py              (번역 검수 알고리즘 개발)
│  ├─ review_ui.py                     (검수 결과를 보여주는 UI 개발)
│  ├─ feedback_system.py               (검수 결과 피드백 기능 추가)
│  └─ database_setup.py                (데이터베이스 설계 및 연동)
├─ templates/                          (HTML 템플릿 파일)
│  ├─ index.html                       (메인 페이지 템플릿)
│  └─ review_results.html              (검수 결과 페이지 템플릿)
├─ static/                             (정적 파일)
│  ├─ css/                             (CSS 파일)
│  │  └─ styles.css                   (스타일 시트)
│  └─ js/                              (JavaScript 파일)
│     └─ scripts.js                   (스크립트 파일)
├─ requirements.txt                   (필요 패키지 목록)
├─ .env.example                       (환경변수 예시 파일)
└─ README.md                          (사용 방법 문서)
```

## 구조 설명
- **translation_review_tool/**: 프로젝트의 루트 폴더입니다.
- **app/**: 애플리케이션의 주요 기능을 구현하는 파일들이 위치합니다.
  - **__init__.py**: 이 폴더를 패키지로 인식하게 하는 초기화 파일입니다.
  - **upload_translation.py**: 번역된 문서를 업로드하는 기능을 구현합니다.
  - **review_algorithm.py**: 번역 검수 알고리즘을 개발하는 파일입니다.
  - **review_ui.py**: 검수 결과를 사용자에게 보여주는 UI를 개발하는 파일입니다.
  - **feedback_system.py**: 검수 결과에 대한 피드백 기능을 추가하는 파일입니다.
  - **database_setup.py**: 데이터베이스를 설계하고 연동하는 파일입니다.
  
- **templates/**: HTML 템플릿 파일들이 위치합니다.
  - **index.html**: 메인 페이지의 HTML 템플릿입니다.
  - **review_results.html**: 검수 결과를 보여주는 페이지의 HTML 템플릿입니다.

- **static/**: 정적 파일들이 위치합니다.
  - **css/**: CSS 파일들이 위치합니다.
    - **styles.css**: 스타일 시트 파일입니다.
  - **js/**: JavaScript 파일들이 위치합니다.
    - **scripts.js**: 클라이언트 측 스크립트 파일입니다.

- **requirements.txt**: 프로젝트에 필요한 패키지 목록을 정의합니다.
- **.env.example**: 환경변수 설정의 예시 파일입니다.
- **README.md**: 프로젝트의 사용 방법 및 설명을 담고 있는 문서입니다.

## 기술 스택
- **Python**: 주 언어로 사용하여 서버 사이드 로직을 구현합니다.
- **Flask**: 경량 웹 프레임워크로, RESTful API 및 웹 애플리케이션을 쉽게 구축할 수 있습니다.
- **HTML/CSS/JavaScript**: 프론트엔드 개발을 위한 기본 기술 스택입니다.
- **SQLite**: 간단한 데이터베이스로, 초기 개발 및 MVP 단계에서 사용하기 적합합니다.

# 누락 검사

### 1. 기능 누락 가능성
- 다양한 파일 형식 지원: PDF, DOCX 외에 다른 형식의 문서 업로드 지원 여부
- 번역 품질 평가 기능: 자동 검수 외에 사용자가 수동으로 검토할 수 있는 기능
- 사용자 인증 및 권한 관리: 여러 사용자가 접근할 수 있는 시스템일 경우 사용자 관리 기능
- 검수 결과의 저장 및 관리 기능: 검수 결과를 데이터베이스에 저장하고 관리하는 기능
- 다국어 지원: 일본어와 한국어 외에 다른 언어에 대한 지원 여부

### 2. 예외처리 누락 가능성
- 파일 업로드 시 오류 처리: 잘못된 형식의 파일이나 크기 초과 파일에 대한 예외 처리
- 검수 알고리즘의 오류 처리: 알고리즘 실행 중 발생할 수 있는 예외 상황에 대한 처리
- 데이터베이스 연결 오류 처리: 데이터베이스와의 연결 실패 시 사용자에게 알림 기능
- 사용자 입력 오류 처리: UI에서 사용자가 잘못된 입력을 했을 때의 예외 처리

### 3. 클라이언트 확인 필요사항
- 검수할 문서의 형식은 무엇인가요? (예: PDF, DOCX 등)
- 검수 기준이나 알고리즘에 대한 구체적인 요구사항이 있나요?
- 사용자 인터페이스에 대한 선호사항이 있나요?
- 검수 결과를 어떻게 저장하고 관리할 것인지에 대한 계획이 있나요?
- 사용자 인증 및 권한 관리에 대한 요구사항이 있나요?
- 검수 결과의 통계 및 리포트 생성 기능에 대한 필요 여부

### 4. 납품 시 필요한 문서
- 사용자 매뉴얼: 시스템 사용 방법에 대한 설명서
- API 문서: 외부 시스템과의 연동을 위한 API 설명서
- 테스트 케이스 및 결과 보고서: 기능 테스트에 대한 문서
- 데이터베이스 스키마 문서: 데이터베이스 설계에 대한 문서
- 코드 주석 및 문서화: 코드에 대한 설명 및 주석

- 다국어 지원 기능 추가

- 기술적 문제: 선택한 기술 스택에서 발생할 수 있는 호환성 문제
- 사용자 경험 문제: UI/UX가 사용자 친화적이지 않을 경우 사용자의 불만 초래
- 프로젝트 일정 지연: 요구사항 변경이나 기술적 문제로 인한 일정 지연 가능성
